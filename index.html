Зачем ?

  1. Моральная - каждый должен иметь возможность пользоваться сайтами.
  2. Юридическая - доступность информации заложена в законодательстве многих стран.
  3. Финансовая - потенциальный клиент.
  4. Имиджевая - компания, которая заботится о доступности своих сайтов - хорошая компания

Критерии доступности сайтов

  • Хорошая контрастность текста и UI компонентов
  • Удобный размер шрифта и UI компонентов
  • Логически понятный интерфейс

Взаимодействие с сайтом с помощью скринридера

  1. Артибут lang=""
  2. Указывайте title для страницы
  3. Атрибут alt="" для изображений
  4. Используйте ARIA - атрибуты
  5. Указывайте тип данных у input
  6. Описывайте input с помощью label
  7. Соблюдайте правильную иерархию заголовков
  8. Старайтесь использовать встроенные html элементы

ARIA

Accessible Rich Internet Applications

  • Роли - для описания типов элемента
  • Атрибуты - для описания состояния и свойств элемента
div roll="alert"
Hello World
/div
Все роли
  • Роли виджетов (button, radio, tab, tooltip, treeitem...)
  • Составние роли (radiogroup, tree, menu, tablist...)
  • Структурные роли (article, heading, group, img...)
  • Landmarks(ориентиры) (main, navigation. region...)
  • У елемента не может быть больше одной роли
  • Не рекомендуется применять противоречащие роли к семантическим элементам
  • Таблица с рекомендация по использованию ролей

Атрибуты ARIA немного отличаются от ролей ARIA. Они также добавляются в разметку, но существует уже определенный диапазон атрибутов ARIA. Все атрибуты используют префикс aria- и делятся на две группы: состояния и свойства.

Атрибуты состояния:

aria-disabled, aria-hidden, aria-invalid, aria-checked, aria-expanded

Атрибуты свойства:

aria-haspopup, aria-autocomplete, aria-label, aria-required, aria-sort, aris-readonly

Live regions:

role="alert"
aria-live="assertive"
aria-live="polite"
